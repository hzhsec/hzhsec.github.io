一、蓝队初级岗位认知

蓝队初级面试整体不算难，但问得最细的一般是：

  • 应急响应
  • 日志分析
  • 安全设备告警分析
  • 常见漏洞原理和 payload
  • WebShell 流量特征

如果时间有限，优先背熟这三块：

  • 应急响应流程
  • 告警研判思路
  • 常见漏洞与流量特征

1. 岗位要求

  • 重点会看各种攻击 payload，熟悉常见漏洞利用流量。
  • 给一个请求包，能判断：
  • 是否为攻击流量；
  • 如果是，属于什么漏洞；
  • 利用思路和危害是什么。
  • 基础能力要求：
  • 基本漏洞扫描和渗透能力；
  • 日志分析能力；
  • 安全设备告警分析能力；
  • 能看懂常见安全产品。

2. 学习重点

  • 多练习常见漏洞利用方法。
  • 学会识别漏洞利用请求和常见 payload。
  • 重点加强：
  • 应急响应；
  • 日志分析；
  • 安全设备告警分析；
  • 流量研判。

四、常见漏洞总复习

1. SQL 注入

原理

用户输入被拼接进 SQL 语句，并被数据库当作代码执行。

成因

  • 开发未做严格过滤；
  • 未使用预编译或参数化查询；
  • 编码与字符集处理不当。

分类

  • 有回显注入
  • 无回显注入
  • 布尔盲注
  • 时间盲注
  • 报错注入
  • 联合注入
  • 堆叠注入
  • 宽字节注入
  • 二次注入
  • Cookie 注入
  • Referer 注入
  • X-Forwarded-For 注入

常见函数

报错注入
  • updatexml()
  • extractvalue()
  • floor()
  • rand()
  • exp()
  • group by
  • count()
  • concat()
盲注
  • if()
  • sleep()
  • substr()
  • substring()
  • left()
  • ascii()
  • ord()
  • length()
  • limit

宽字节注入

  • 常见场景：
  • Web 使用 UTF-8；
  • 数据库使用 GBK。
  • 由于 GBK 为双字节编码，转义符可能和前字节重新组合，导致单引号逃逸。
  • 常被提到的函数：
  • magic_quotes_gpc()

堆叠注入

  • MySQL 默认用分号 ; 结束语句；
  • 一条无害语句后继续拼接恶意语句，实现多条 SQL 执行。

Dnslog 注入

  • 借助 load_file() 读取共享路径；
  • 拼接 DNSLog 域名，把数据外带。

联合注入步骤

  1. 找传参点
  2. 判断闭合符
  3. 判断列数
  4. 判断显示位
  5. 查询 database()
  6. 查表
  7. 查字段和数据

判断闭合符常见方式

  • id=1 and 1=1
  • id=1' and 1=2
  • id=1" or 1=1
  • ') or 1=2
  • and 234=234
  • and 1
  • or 1
  • and 1^1
  • &amp;&amp;1
  • ||0

过滤了怎么办

  • 关键字过滤：
  • 大小写绕过；
  • 双写绕过；
  • URL 编码绕过。
  • 特殊字符或空格过滤：
  • URL 编码；
  • 特殊空白字符。
  • 注释过滤：
  • 闭合绕过；
  • 逻辑运算绕过。
  • 单引号过滤：
  • 宽字节绕过。
  • 明文参数过滤：
  • Burp 进行业务层编码后重放。

空格替代

  • /**/
  • /!/
  • +
  • %09
  • %0a
  • %0b
  • %00
  • %a0
  • 括号

关键字和函数绕过

  • 16 进制
  • char()
  • 字符串拼接
  • sleep() -> benchmark()
  • if() -> case when then else end
  • ascii() -> ord()
  • substr() -> substring()
  • 内联注释：
  • /! ... /

写 WebShell / 提权

写文件
  • 需要写权限。
写日志
  • 不一定需要直接网站权限；
  • 需要能开启日志并指定日志路径到网站目录。
日志写 Shell 示例
select '一句话木马' into outfile '路径';
select '一句话木马' into dumpfile '路径';

前提：

  • Root 权限；
  • GPC 关闭；
  • 已知文件路径。

拿到 SQL 权限后如何拿系统权限

UDF 提权
  • 原理：导出可调用 cmd 的 udf.dll 动态链接库。
  • 条件：
  • secure_file_priv='' 或满足写文件条件；
  • 账户权限高，最好是 Root；
  • 对安装目录有写权限。
MOF 提权
  • 常见于 Windows Server 2003。
  • C:/Windows/system32/wbem/mof/ 目录中的 .mof 文件会被系统周期性执行，可借此执行命令。

防御

  • 预编译
  • 参数化查询
  • 白名单限制
  • 敏感字符过滤
  • 编码统一
  • 站库分离

面试常问

MySQL 5.0 上下区别
  • 5.0 以下没有 information_schema；
  • 常需暴力跑库跑表。
为什么一个站点只有 80 端口开放
  • 数据库端口改了；
  • 站库分离；
  • 3306 没对外开放。

Sqlmap 常用参数

  • -r
  • -u
  • -p
  • *
  • -m
  • --os-shell
  • --os-cmd
  • --tamper
  • --level
  • --risk
  • --random-agent
  • --batch
  • --dbms

2. XSS

原理

通过注入恶意 HTML 或 JavaScript，最终攻击浏览器。

类型

  • 反射型
  • 存储型
  • DOM 型

区别

  • 反射型：后端处理后回显到页面。
  • DOM 型：前端 JavaScript 直接修改 DOM 触发。

常见 JavaScript 函数

  • document.cookie
  • console.log('xss')

绕过方式

  • 大小写绕过
  • javascript: 伪协议
  • 不写分号
  • Flash
  • HTML5 新标签
  • Fuzz 测试
  • 双层标签绕过

常见工具

  • xsstrike

XSS 平台与利用

  • BEEF
  • xss.pt

危害

  • 窃取 Cookie
  • 会话劫持
  • 浏览器挂钩
  • 钓鱼跳转
  • 账户接管

XSS 蠕虫产生条件

只要页面存在用户间交互，就可能形成蠕虫，不一定必须是存储型。

有 Shell 的情况下如何长久控制

  • 在后台登录处插入记录账号密码的恶意 JS；
  • 在登录后页面植入存储型 XSS。

防御

  • 输出时做实体编码；
  • 过滤危险事件；
  • 限制输入长度；
  • 使用 HttpOnly；
  • 开启 CSP；
  • 及时修补漏洞。

3. XXE

原理

  • PHP 开启外部实体引用；
  • 传入 XML 数据未做过滤。

特点

  • 参数中常出现 XML 标签；
  • 响应包 Content-Type 常为 text/xml。

利用

  • file:// 读取文件
  • http://127.0.0.1:22 探测端口
  • PHP 伪协议读取文件
  • 盲 XXE：
  • 加载远程 XML；
  • 做数据外带。

防御

  • 关闭外部实体功能；
  • 严格校验 XML 输入。

4. CSRF

原理

攻击者利用用户已登录状态，诱导浏览器向目标站点自动发起请求。

攻击过程

  1. 构造恶意请求
  2. 放到链接、图片、表单中
  3. 用户在登录态访问后自动发出请求

防御

  • 关键操作二次认证
  • Token 校验
  • 验证 Referer / Origin
  • HttpOnly

5. SSRF

原理

服务端提供从其他服务器获取数据的能力，但未对目标地址做限制，攻击者可借此让服务器替自己发请求。

常见场景

  • 加载图片
  • 拉取网页
  • 下载远程资源
  • 邮件系统
  • 文件处理
  • 在线处理工具

支持协议

  • http://
  • https://
  • dict://
  • ftp://
  • file://
  • php://
  • gopher://

常见危险函数

  • file_get_contents()
  • fsockopen()
  • curl_exec()

危害

  • 内外网端口和服务扫描
  • 本地敏感文件读取
  • 利用内网服务漏洞
  • 探测内外网 Web 服务

绕过

  • 利用 @
  • 添加端口号
  • 短链接
  • IP 进制转换
  • DNS 解析绕过

防御

  • 输入校验
  • 白名单访问
  • 禁用危险协议
  • 限制访问内网地址

6. 文件上传

原理

上传点对文件类型、内容、解析方式校验不严，攻击者可上传恶意脚本并在服务端执行。

中间件解析漏洞

Apache
  • 多后缀解析
  • 罕见后缀解析
  • .htaccess 解析
  • 例如：
  • 1.php.rar
IIS
  • 分号截断：
  • 1.asp;.jpg
  • 罕见后缀：
  • .asa
  • .cer
  • .cdx
  • 文件夹解析：
  • 1.asp/1.jpg
  • 1.asa/1.jpg
  • 1.cer/1.jpg
  • CGI 解析漏洞
  • PUT 上传漏洞
Nginx
  • CGI / cgi.fix_pathinfo 解析问题
  • 如：
  • /1.jpg/1.php

黑名单绕过

  • 特殊后缀：
  • .php3
  • .php4
  • .php5
  • .phtml
  • 上传 .htaccess
  • 上传 .user.ini
  • 大小写绕过
  • 后缀前加空格
  • 文件名以点结尾
  • ::$DATA
  • 双写后缀：
  • 1.pphphp
  • 双点：
  • x.php..

白名单绕过

  • %00 截断
  • 图片马
  • 条件竞争
  • 配合文件包含

内容绕过

  • 填充垃圾字符
  • 免杀

前端绕过

  • 关闭前端 JS
  • Burp 改包

防御

  • 服务端校验
  • 白名单校验
  • 校验 MIME
  • 校验文件头与完整性
  • 上传目录不可执行
  • 上传目录与 Web 服务隔离
  • 重命名文件
  • 图片重渲染
  • 重建文件内容
  • 检查文件内容

7. 文件包含 / 下载 / 目录遍历

文件包含原理

应用动态包含文件时未校验用户输入，导致攻击者可包含本地文件、远程文件或图片马。

常见函数

  • include()
  • include_once()
  • require()
  • require_once()

协议

  • file://
  • http://
  • https://
  • php://filter
  • php://input
  • data://
  • zip://
  • phar://
  • zlib://
  • glob://
  • ssh2://
  • rar://
  • ogg://
  • expect://

利用

  • ../ 目录遍历
  • 配合文件上传获取 WebShell
  • 读取源码
  • 读取敏感文件

文件下载漏洞原理

攻击者通过构造请求，下载本不该公开访问的敏感文件，例如配置文件、账号密码、业务数据。

目录遍历原理

服务器配置不当或文件名校验不足，导致可访问 Web 根目录以外文件，严重时还能配合其他漏洞执行命令。

防御

  • 过滤 .、\、/
  • 使用 open_basedir
  • 禁止远程文件包含
  • 关闭 allow_url_fopen / allow_url_include
  • 对下载参数做访问控制和合法性校验

8. 命令执行 / 代码执行

命令执行漏洞原理

攻击者向程序提交恶意输入，程序拼接后调用系统命令执行。

代码执行漏洞原理

应用把用户输入当作代码解释执行，最终导致任意代码运行。

防御

  • 严格输入校验
  • 过滤危险字符
  • 最小权限运行
  • 正确设置文件和目录权限

9. 序列化与反序列化

PHP 反序列化

  • 用户可控数据进入 unserialize()；
  • 类中存在可利用魔术方法时，可能导致代码执行、目录遍历、SQL 注入等。

常见函数

  • serialize()
  • unserialize()

常见魔术方法

  • __wakeup()
  • __sleep()
  • __construct()
  • __destruct()
  • __call()
  • __callStatic()

补充

  • 某些场景下，成员属性数大于实际数目，可用于绕过 __wakeup() 限制。

Java 反序列化

原理

序列化是把对象转成二进制，反序列化是再还原为对象。若输入对象是恶意构造的，反序列化时就可能执行恶意逻辑。

修复
  • 升级组件
  • 打补丁
  • 限制危险类和反序列化入口
常见题
  • Shiro：
  • RememberMe
  • 默认 AES Key 可被利用伪造恶意数据
  • Fastjson：
  • @type 指定类名
  • 恶意 JSON 可导致代码执行
  • Log4j：
  • ${jndi:...}
  • 配合 LDAP/RMI 远程加载

八、WebShell 与木马流量特征

1. 菜刀

  • 早期多为明文，后常见 Base64。
  • 常见参数：
  • z0
  • z1
  • z2
  • 常见执行函数：
  • eval
  • 少量情况可能为 assert
  • 参数接收可能来自：
  • $_POST
  • $_GET
  • $_REQUEST

2. 蚁剑

  • 默认 User-Agent 常见 antsword
  • 常见特征代码：
  • @ini_set("display_errors","0");
  • 混淆后参数名常为：
  • _0x...

3. 冰蝎

通用特征

  • 常为 POST 请求；
  • Content-Type: application/octet-stream 为强特征；
  • 默认 Accept 与默认 UA 偏旧，属弱特征。

冰蝎 2

  • 默认 Accept 值较特殊；
  • 默认内置 UA 偏旧；
  • Content-Length: 16 常被作为连接特征。

冰蝎 3

  • 取消动态密钥获取；
  • 常见 POST；
  • application/octet-stream 仍值得重点关注；
  • 默认 UA 也较旧，但可以修改。

4. 哥斯拉

  • 常为加密流量；
  • 默认 User-Agent 可能带 Java 版本信息。

常见特征：

  • 发送固定代码后 HTTP 响应为空；
  • 发送固定测试代码，结果固定；
  • 存在如 getBasicsInfo 的固定交互阶段。

5. WebShell 简介

WebShell 是以 asp、php、jsp、cgi 等形式存在的后门或远程管理代码，可用于文件管理、命令执行、数据库管理、权限维持等。

4. 常见问答

天眼可以捕捉到 CMD 命令吗

可以。尤其是远控内网主机并回显执行结果时，可能在流量中体现。

内网横向常见告警类型

  • CS 相关
  • 隧道类
  • 内网扫描
  • 暴力破解

如何判断资产是否失陷

  • 持续外联恶意地址；
  • 存在 Shell 或隧道连接；
  • 内网主机持续攻击其他内网资产。

受害 IP 为源 IP 说明什么

可能说明主机已被控后主动对外通信，也可能存在 IP 欺骗，要结合上下文分析。

如何搜索端口

  • sport eq 80
  • sport eq 80 or sport eq 443

如何搜索源 IP 为 A、目的 IP 为 B

ip(A) AND dip(B)

说明：

  • 运算符要大写：
  • AND
  • OR
  • NOT

目的 IP 是 114.114.114.114，端口是 53，要不要封

不能直接封。这个通常是 DNS 地址，应先确认真实受害资产和上下文。

不出网主机通过哪种代理建立连接

  • 常见回答：正向代理。

正向代理与反向代理

  • 正向代理：代理客户端，服务端不知道真实客户端。
  • 反向代理：代理服务端，客户端不知道真实服务端。

天眼小工具能做什么

  • 常见编码解码，如：
  • Base64
  • URL

主机外联排查思路

  • 判断外联目标是否恶意；
  • 是否为 DNSLog 平台；
  • 是否高频请求；
  • 是否命中威胁情报。

告警成功如何处理

  • 先判断是否真的利用成功；
  • 结合回显特征和数据包分析；
  • 若无法仅凭数据包判断，可尝试复现；
  • 确认成功后，立即同步研判组或客户。
