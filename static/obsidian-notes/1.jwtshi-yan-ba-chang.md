## 四.通过JWK头注入实现JWT认证绕过

**靶场地址**:
[实验：通过JWK头注入实现JWT认证绕过 |Web Security Academy](https://portswigger.net/web-security/jwt/lab-jwt-authentication-bypass-via-jwk-header-injection)
**原理:**
`JWK`内嵌公钥攻击,就是`jwt`将公钥放在头部,攻击者通过修改头部控制 `JWT `验证所使用的公钥,从而使攻击者变成了签发` JWT` 的人.

`JWT header `允许：
```json
{  
 "alg":"RS256",  
 "jwk":{  
   "kty":"RSA",  
   "n":"攻击者公钥",  
   "e":"AQAB"  
 }  
}
```
如果服务器：
```
直接使用 header.jwk 验证
```
攻击者：
- 用自己私钥签名
- 把公钥放 header
服务器验证成功。

**复现:**

1.获取带有自己`jwt`的`/admin`包发送到`repeater`模块,使用`jwt editor`生成新的`rsa key`
![image.png](https://cdn.jsdmirror.com/gh/hzhsec/upload@main/20260310215427231.png)

![image.png](https://cdn.jsdmirror.com/gh/hzhsec/upload@main/20260310215449004.png)

成功生成
![image.png](https://cdn.jsdmirror.com/gh/hzhsec/upload@main/20260310215514128.png)

2.在`repeater`模块,注入自己的`rsa公钥`
![image.png](https://cdn.jsdmirror.com/gh/hzhsec/upload@main/20260310215708732.png)

3.修改用户之后还要点一下`sign`进行签名
![image.png](https://cdn.jsdmirror.com/gh/hzhsec/upload@main/20260310215914085.png)

![image.png](https://cdn.jsdmirror.com/gh/hzhsec/upload@main/20260310215935442.png)

==注意==:如果生成过多个key的话,要选择对应的,不要选错了

4.发送数据,已经变成200,可以访问了,可以直接删除用户了

**原理:**
与上一个类似,上一个是直接使用头部的公钥进行验证,这一个则是头部使用`jku`字段,加载远程的公钥字段,攻击者只需要搭建一个远程服务器构造特定的返回公钥值,就可以实现绕过认证.

`JWT header `支持字段：
```json
{
  "alg": "RS256",
  "jku": "https://attacker.com/jwks.json",
  "kid": "hack"
}
```
`jku` = JSON Web Key URL
含义:指定` JWKS` 地址

`JWKS`格式:
```json
{  
"keys":[  
	{  
	"kid":"hack",

	}  
   ]  
}
```

## 六. 通过kid头路径遍历实现JWT认证绕过

**靶场地址**:
[实验：通过kid头路径穿越JWT认证 |网络安全学院](https://portswigger.net/web-security/jwt/lab-jwt-authentication-bypass-via-kid-header-path-traversal)

**原理**:
有些代码写成
```
key = open("/keys/" + kid + ".pem")
```
通过头部的`kid`遍历本地的签名文件来作为jwt的签名
攻击者可以通过使用`../../`使服务器读取其他目录的文件,进行签名,然而服务器根目录`/dev/null`会返回空
即可以使用这个来用空来==验签==,我们用空来==签名==

**复现**:
1.一如既往登录将数据包发送到`repeater`,使用`jwt editor` 生成`symmetric key`
![image.png](https://cdn.jsdmirror.com/gh/hzhsec/upload@main/20260310224219491.png)


2.将k改成`AA==`,`AA==`是空的`base64编码`
![image.png](https://cdn.jsdmirror.com/gh/hzhsec/upload@main/20260310224213326.png)

3.回到`repeater`模块的`json web token`,将kid改成`../../../../../../../dev/null`,跳出到根目录的`/dev/null`,`administrator`记得修改
![image.png](https://cdn.jsdmirror.com/gh/hzhsec/upload@main/20260310224556319.png)

4.签名jwt,点`Sign`选择刚刚生成的key,确认

![image.png](https://cdn.jsdmirror.com/gh/hzhsec/upload@main/20260310224658267.png)

5,成功200
![image.png](https://cdn.jsdmirror.com/gh/hzhsec/upload@main/20260310224825320.png)
