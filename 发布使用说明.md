# 发布使用说明

本文档说明 [publish_obsidian.py](/F:/hugo/hzhsec.github.io/scripts/publish_obsidian.py) 的常用发布方式。

## 脚本位置

```powershell
F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py
```

## 默认站点目录

脚本默认会把站点文件生成到：

```powershell
F:\hugo\hzhsec.github.io
```

也就是说，就算你在别的目录执行命令，只要不额外传 `--site-root`，生成结果也会写回这个仓库，不会写到你当前终端所在目录。

## 常用命令

### 1. 单篇预览

只检查将要发布成什么路径，不真正写入站点：

```powershell
python F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py .\文章.md --dry-run
```

### 2. 单篇发布

发布一篇文章并重建本地站点，但不提交 git：

```powershell
python F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py .\文章.md
```

### 3. 批量预览

预览当前目录下所有 `.md` / `.markdown` 文件会发布到哪里：

```powershell
python F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py .\ --batch --dry-run
```

### 4. 批量发布

批量发布当前目录下所有 `.md` / `.markdown` 文件，并重建本地站点，但不提交 git：

```powershell
python F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py .\ --batch
```

### 5. 发布后自动提交并推送

如果你确认内容没问题，再加 `--git-push`：

```powershell
python F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py .\ --batch --git-push
```

如果想自定义提交信息：

```powershell
python F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py .\ --batch --git-push --commit-message "batch publish CVE notes"
```

### 6. 只重建站点

不读取新文章，只按当前状态重建整站：

```powershell
python F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py --rebuild-only
```

## 输出说明

脚本输出里常见状态含义：

- `NEW PREVIEW`：新文章预览，还没有真正写入站点。
- `UPDATE PREVIEW`：旧文章将被更新，还没有真正写入站点。
- `NEW DONE`：新文章已经发布并写入本地站点。
- `UPDATE DONE`：旧文章已经被新内容覆盖更新。
- `TOTAL: N`：本次处理了多少篇文章。
- `BATCH DONE: N`：本次批量发布完成，共处理 N 篇。

## 更新判定规则

脚本会优先按文章最终路径判断是“新建”还是“更新”：

- 同一个 `rel_permalink`：视为更新。
- 如果路径没重复，但 `source_note` 能对上旧记录：也视为更新。
- 其他情况：视为新建。

简单理解：

- 同栏目、同 slug、同路径：会更新旧文章。
- 如果你改了栏目或 slug：大概率会被当成新文章。

## 备份说明

默认发布后会把源 Markdown 备份到：

```powershell
F:\hugo\hzhsec.github.io\static\obsidian-notes
```

如果不想生成备份，可以加：

```powershell
--no-backup
```

## 手动指定站点目录

虽然脚本默认已经指向仓库根目录，但你也可以显式指定：

```powershell
python F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py .\ --batch --site-root F:\hugo\hzhsec.github.io
```

## 推荐使用顺序

推荐平时按这个顺序走：

1. 先预览

```powershell
python F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py .\ --batch --dry-run
```

2. 确认路径没问题后，本地发布

```powershell
python F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py .\ --batch
```

3. 本地检查站点没问题后，再决定要不要提交

```powershell
python F:\hugo\hzhsec.github.io\scripts\publish_obsidian.py .\ --batch --git-push
```
